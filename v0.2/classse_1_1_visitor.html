Detailed Description

template<typename T>
interface se::Visitor< T >

Traversal algorithm for an acyclic directed graph (DAG)

Visitor can traverse a DAG whose vertices are Expr subclasses. Recall that this DAG represents the syntactic structure of a symbolic expression. The order of the traversal is implemention-specific.

Let E be an Expr subclass such that E::kind() < EXT_EXPR. Then, E is traversable by Visitor<T>::visit(const E&). Otherwise, E is visited by visit(const Expr&). This eases the incremental extension with new Expr subclasses.

Note that alternative designs are discussed in the object-oriented programming literature in the context of the "Expression Problem".
