Detailed Description

Acceptor in Visitor pattern.

Must be implemented by every Expr subclass to support polymorphic DAG traversals with the Visitor interface. To facilitate this, the Expr class inherits Walker. Since it declares only pure virtual member functions with the WALK_DECL macro, each Expr subclass should use WALK_DEF inside its class to define the required walk member functions.

Example: 

class UnaryExpr : public Expr {
  public:
  ...
  WALK_DEF(T1)
  WALK_DEF(T2)
  ...
  WALK_DEF(TN)
}

where T1 ... TN are types for which there exists a Visitor<T> template class.
