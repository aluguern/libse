Detailed Description

Untyped read or write event.

An Event object can only be instantiated through a subclass. Usually, these objects should be made accessible through a shared smart pointer such as std::shared_ptr because during the analysis events often need to be put in relation to each other.

An axiom is that two events `e` and `e'` are equal if and only if both their heap references are identical, i.e. `&e == &e'`. To enforce this equality axiom for subclasses as well, each Event object has a unique identifier.

An event guarded by a non-null condition is said to be conditional; otherwise, it is said to be unconditional.
